# EditorialWorkflow

The editorial workflow should optimize for correctness, traceability, and incremental depth.

## Core Principle

The first milestone is not scale.
The first milestone is a complete specification for one poem.

Once one poem can be represented rigorously, the workflow can be repeated across the corpus.

## Workflow Stages

### 1. Source Intake

- identify poem
- verify source text
- identify edition or source basis

### 2. Base Entry

- create poem identity record
- add original text
- add basic translation or transliteration

### 3. Annotation Pass

- add word meanings
- add key ontology tags
- add initial commentary

### 4. Evidence And Citation Pass

- classify major claims by evidence class
- attach citations
- note ambiguity or disagreement

### 5. Review Pass

- editorial review
- scholarly review where needed
- consistency review against ontology

### 6. Quality Assignment

- assign coverage level
- assign editorial quality level
- publish or return for revision

## Ongoing Rule

Editorial work should move a poem toward completeness in a controlled way rather than
trying to make every poem Diamond-level on first entry.
