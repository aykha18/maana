# PoemPage

## Purpose

The poem page is the main reading surface. It should support serious reading while
gently preparing the user where necessary.

## Page Structure

### 1. Header

- poem title
- poet
- language
- era
- estimated reading time

### 2. Foundations for Understanding

Optional block shown above commentary and below core poem metadata.

Example language:

"To appreciate this poem more deeply, familiarity with these ideas will help."

Each foundation item should include:

- label
- short why-it-matters note
- readiness state
- click target to concept, experience, or lesson

### 3. Poem Text

- original
- transliteration when relevant
- translation

### 4. Commentary

Commentary remains important, but it should no longer be the first or only bridge
to meaning.

### 5. Related Concepts and Experiences

Show:

- primary experience
- secondary experiences
- key concepts
- symbols

### 6. Reflection

Optional prompt such as:

- Have you ever waited for something that never came?
- Which image in this poem changed the way you understand longing?

### 7. Return Loop

If the user visited a foundation or lesson before opening the poem, the page should
acknowledge that context and invite rereading.

Example:

"You recently explored longing. Notice how absence shapes the poem's emotional logic."

## UX Rules

- Foundations must never block access to the poem.
- The poem must remain the primary visual object.
- Reflection is optional and private by default.
- Readiness language must be gentle and non-academic.
