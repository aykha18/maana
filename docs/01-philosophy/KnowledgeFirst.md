# KnowledgeFirst

The product should be designed from meaning outward.

Most content products begin by designing screens, endpoints, and workflows. World Poetry
Explorer should begin by defining its concepts and relationships. The application exists to
express a knowledge model, not to hide the absence of one.

## What Knowledge-First Means

- Define the domain before building features.
- Define ontology before choosing database tables.
- Define editorial standards before writing commentary at scale.
- Define relationships before building recommendations.
- Define meaning before adding AI.

## Why This Matters

Poetry is not only text. It includes context, ambiguity, symbolism, history,
translation, and layered interpretation. If those structures are not documented,
implementation will become inconsistent.
